# Planning Documents

Use `docs/planning/` for active OpenForge planning material such as:

- discovery plans
- delivery sequencing
- phase closeout notes
- risk research
- architecture options
- schema planning
- workbook deconstruction planning
- MVP scoping notes
- UI/UX working interpretation notes

Keep these documents practical and current. When a planning note becomes superseded, mark it clearly instead of silently relying on old versions.

Current profile-scoped OpenForge planning documents take precedence over older archived references.

Suggested phase grouping:

- `Phase 1`: workbook deconstruction, parity findings, and contract drafting
- `Phase 2`: profile-scoped architecture, schema planning, and calculation-structure planning
- `Pre-Code`: stack, scaffold, branch, and first-slice planning
- later phases: implementation slices after explicit approval
